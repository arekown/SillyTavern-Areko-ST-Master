# Areko-ST-Master

A flexible, fully customizable **JSON data tracker** for roleplay in [SillyTavern](https://github.com/SillyTavern/SillyTavern) — inspired by WTracker, but rebuilt from scratch with a visual schema builder, a slide-in companion panel, and per-character image generation.

> Ein flexibler, voll anpassbarer **JSON-Datentracker** für Rollenspiele in SillyTavern — inspiriert von WTracker, aber von Grund auf neu gebaut, mit visuellem Schema-Builder, einem Slide-in-Panel und Bildgenerierung pro Charakter.

---

## 🇬🇧 English

### What it does

Areko-ST-Master watches your roleplay and keeps a structured tracker (time, location, weather, and a rich per-character state: appearance, clothing, inventory, condition, the three distinct mind layers, relationships, quests, skills). The tracker is generated by your own LLM, stored per message, shown above each message, fed back into the chat so the model stays consistent, and surfaced in a companion panel with character portraits.

### Features

- **Visual schema builder (CMS).** Build your own tracker structure: categories and fields with types *Text, Number, Yes/No, List, Group, Object-List*. Per field: description (the instruction the LLM follows), enum values, example, default, min/max, display style (*text / chip / badge / bar*), and an *applies to* rule (everyone / NPCs only / player only).
- **Global vs. per-character categories.** A global category is one object (e.g. the scene). A per-character category loops over every participating character — one entry each.
- **Two default presets out of the box.** *Default DE* (German labels) and *Default EN* (English labels). The field instructions stay English for model quality; only the labels differ.
- **Preset management.** Create, duplicate, rename, reset to default, export and import presets as JSON.
- **Full chat context.** Generation sends the same context a normal chat turn gets (character card, persona, world info, instruct formatting, last *X* messages) plus the most recent tracker(s).
- **Prompt-engineered JSON.** Provider-agnostic — works on any backend, no native JSON-schema support required.
- **Editable prompts.** Three prompts you can edit and reset to default: the tracker behavior prompt (`{{language}}`), the JSON-format prompt (`{{schema}}`, `{{example_response}}`), and the image prompt (`{{name}}`, `{{details}}`).
- **In-chat tracker.** A generate / regenerate / delete control and a collapsible tracker block above every message. Trackers are saved in the message and survive a reload.
- **Auto mode.** Generate automatically after AI responses (and/or after your inputs). Image and empty messages are never tracked (the `/sd` guard).
- **Companion panel.** A slide-in panel on the right with three tabs — **General**, **Player**, **NPCs**. The player is detected via your persona name; everyone else is an NPC. Groups and NPC cards are collapsible.
- **Per-character images.** Generate a portrait via `/sd` (uses your configured image backend), upload your own, or delete it. Images are named after the character. Click a portrait to enlarge it full-screen.

### Installation

In SillyTavern: open **Extensions → Install Extension**, then paste this URL:

```
https://github.com/arekown/SillyTavern-Areko-ST-Master
```

Then reload SillyTavern. (Requires an image backend configured in SillyTavern if you want portrait generation.)

### How to use it

1. **Pick a connection profile.** Open the **Extensions** settings (the gear/puzzle panel) → **Areko-ST-Master → Open settings**. On the **General** tab, choose a *Connection Profile* (set one up in SillyTavern's Connection Manager first).
2. **Choose your preset & language.** On the **Layout** tab, pick *Default DE* or *Default EN* from the profile bar, or build your own structure. Set the UI language on the **General** tab.
3. **Generate a tracker.** Click **Generate tracker** above any message, or turn on **Auto mode** (*On responses only* is the common choice) so it runs after every AI reply.
4. **Open the panel.** Click the **Areko-ST-Master** tab on the right edge of the screen. Switch between *General / Player / NPCs*.
5. **Add portraits.** Expand a character card in the panel and use the buttons (top-left: generate + upload; below: delete). Click a portrait to view it full-size.
6. **Tune the prompts.** On the **Prompts** tab you can edit and reset the tracker, JSON-format, and image prompts.

## 🇩🇪 Deutsch

### Was es macht

Areko-ST-Master beobachtet dein Rollenspiel und führt einen strukturierten Tracker (Zeit, Ort, Wetter und einen reichen Zustand pro Charakter: Aussehen, Kleidung, Inventar, Verfassung, die drei getrennten „Geist"-Ebenen, Beziehungen, Quests, Fähigkeiten). Der Tracker wird von deinem eigenen LLM erzeugt, pro Nachricht gespeichert, über jeder Nachricht angezeigt, in den Chat zurückgegeben (damit das Modell konsistent bleibt) und in einem Panel mit Charakter-Porträts dargestellt.

### Funktionen

- **Visueller Schema-Builder (CMS).** Bau deine eigene Tracker-Struktur: Kategorien und Felder mit den Typen *Text, Zahl, Ja/Nein, Liste, Gruppe, Objekt-Liste*. Pro Feld: Beschreibung (die Anweisung, der das LLM folgt), feste Auswahl (Enum), Beispiel, Standardwert, Min/Max, Darstellung (*Text / Chip / Badge / Balken*) und eine *Gilt-für*-Regel (alle / nur NPCs / nur Spieler).
- **Globale vs. Pro-Charakter-Kategorien.** Eine globale Kategorie ist ein Objekt (z. B. die Szene). Eine Pro-Charakter-Kategorie wird über alle beteiligten Charaktere geloopt — ein Eintrag pro Charakter.
- **Zwei Default-Profile von Haus aus.** *Default DE* (deutsche Labels) und *Default EN* (englische Labels). Die Feld-Anweisungen bleiben für die Modellqualität englisch; nur die Labels unterscheiden sich.
- **Profil-Verwaltung.** Profile anlegen, duplizieren, umbenennen, auf Default zurücksetzen, als JSON exportieren und importieren.
- **Voller Chat-Kontext.** Die Generierung schickt denselben Kontext mit wie eine normale Chat-Antwort (Charakterkarte, Persona, World-Info, Instruct-Formatierung, letzte *X* Nachrichten) plus die letzten Tracker.
- **Prompt-engineered JSON.** Providerunabhängig — läuft auf jedem Backend, keine native JSON-Schema-Unterstützung nötig.
- **Editierbare Prompts.** Drei Prompts, die du bearbeiten und zurücksetzen kannst: der Tracker-Verhaltens-Prompt (`{{language}}`), der JSON-Format-Prompt (`{{schema}}`, `{{example_response}}`) und der Bild-Prompt (`{{name}}`, `{{details}}`).
- **Tracker im Chat.** Über jeder Nachricht ein Generieren-/Neu-generieren-/Löschen-Knopf und ein aufklappbarer Tracker-Block. Tracker werden in der Nachricht gespeichert und überleben einen Reload.
- **Auto-Modus.** Automatisch nach KI-Antworten generieren (und/oder nach deinen Eingaben). Bild- und Leer-Nachrichten werden nie getrackt (`/sd`-Schutz).
- **Begleit-Panel.** Ein Slide-in-Panel rechts mit drei Registern — **Allgemein**, **Spieler**, **NPCs**. Der Spieler wird über deinen Persona-Namen erkannt; alle anderen sind NPCs. Gruppen und NPC-Karten sind auf-/zuklappbar.
- **Bilder pro Charakter.** Ein Porträt per `/sd` erzeugen (nutzt dein konfiguriertes Bild-Backend), ein eigenes hochladen oder löschen. Bilder werden nach dem Charakternamen benannt. Klick auf ein Porträt öffnet es als Vollbild.

### Installation

In SillyTavern: **Erweiterungen → Erweiterung installieren** öffnen und diese URL einfügen:

```
https://github.com/arekown/SillyTavern-Areko-ST-Master
```

Danach SillyTavern neu laden. (Für die Porträt-Generierung muss in SillyTavern ein Bild-Backend eingerichtet sein.)

### Bedienung

1. **Connection-Profil wählen.** Im **Erweiterungen**-Panel (Zahnrad) → **Areko-ST-Master → Einstellungen öffnen**. Im Register **Allgemeines** ein *Connection-Profil* wählen (vorher im Connection-Manager von SillyTavern anlegen).
2. **Profil & Sprache wählen.** Im Register **Layout** in der Profil-Leiste *Default DE* oder *Default EN* wählen — oder eine eigene Struktur bauen. Die Oberflächensprache stellst du im Register **Allgemeines** ein.
3. **Tracker generieren.** Über einer Nachricht auf **Tracker generieren** klicken, oder den **Auto-Modus** aktivieren (*Nur bei Antworten* ist die übliche Wahl), damit er nach jeder KI-Antwort läuft.
4. **Panel öffnen.** Am rechten Bildschirmrand auf den **Areko-ST-Master**-Griff klicken. Zwischen *Allgemein / Spieler / NPCs* wechseln.
5. **Porträts hinzufügen.** Im Panel eine Charakter-Karte aufklappen und die Buttons nutzen (oben links: Generieren + Hochladen; darunter: Löschen). Klick aufs Porträt zeigt es in Vollbild.
6. **Prompts anpassen.** Im Register **Prompts** kannst du den Tracker-, JSON-Format- und Bild-Prompt bearbeiten und zurücksetzen.

## 🗺️ Roadmap — ComfyUI integration with appearance fidelity / ComfyUI-Integration mit Aussehens-Treue

> Goal: portraits that actually keep the same face/look for a given character across regenerations — not just a fresh random person each time. / Ziel: Porträts, die das Gesicht/den Look eines Charakters über Neu-Generierungen hinweg wirklich beibehalten — nicht jedes Mal eine neue Zufallsperson.

### Status quo

Today, **Generate** sends a prompt assembled from the character's appearance fields to `/sd`, which routes to whatever image backend SillyTavern is configured with — **ComfyUI already works** as that backend. What's missing is *consistency*: the same character should look the same every time. / Aktuell baut **Generieren** einen Prompt aus den Aussehens-Feldern und schickt ihn an `/sd`, das an das in SillyTavern eingestellte Bild-Backend geht — **ComfyUI funktioniert hier schon**. Was fehlt, ist *Konsistenz*.

### Plan (phased / in Phasen)

**Phase 1 — Determinism from data we already have / Determinismus aus vorhandenen Daten**
- Per-character **seed lock** (store a fixed seed per name; reuse it on regenerate).
- Per-character **prompt prefix / negative prompt** fields (e.g. a trigger word, style, "consistent face").
- Build the positive prompt from the chosen appearance field/group (already implemented) and snap to a portrait resolution.
- *DE:* Pro Charakter **Seed-Lock**, **Prompt-Präfix/Negativ-Prompt**; Positiv-Prompt aus dem gewählten Aussehens-Feld/-Gruppe; Auflösung auf Porträt einrasten.

**Phase 2 — Reference-image fidelity via a custom ComfyUI workflow / Referenzbild-Treue über einen eigenen ComfyUI-Workflow**
- Use the first generated (or uploaded) portrait as a **reference image** stored per character.
- Ship a ComfyUI workflow template that feeds that reference through **IPAdapter / InstantID / FaceID** (face & style conditioning) so new generations keep the same face while the prompt still drives pose, outfit and scene.
- Drive it through SillyTavern's ComfyUI "custom workflow" mechanism, injecting the reference image and the appearance prompt into the workflow's input nodes.
- *DE:* Erstes erzeugtes (oder hochgeladenes) Porträt als **Referenzbild** pro Charakter speichern; ComfyUI-Workflow-Vorlage mit **IPAdapter / InstantID / FaceID** ausliefern; Referenzbild + Aussehens-Prompt über STs „Custom-Workflow"-Mechanismus in die Eingabe-Knoten einspeisen.

**Phase 3 — Robustness & convenience / Robustheit & Komfort**
- Optional **ReActor face-swap** as a fallback: generate freely, then swap the locked face on top.
- Per-character **LoRA / embedding** field for trained likenesses.
- A small **gallery** per character (swipe between past generations, pick a new reference).
- Batch "regenerate all present characters".
- *DE:* Optionaler **ReActor-Face-Swap** als Fallback; pro Charakter ein **LoRA-/Embedding**-Feld; kleine **Galerie** pro Charakter; Stapel-„alle anwesenden Charaktere neu generieren".

### Open questions to verify before building / Offene Punkte vor dem Bau

- The exact surface SillyTavern exposes for **custom ComfyUI workflows** via `/sd` (how to inject a dynamic reference image and prompt into named workflow nodes). / Wie genau SillyTavern **Custom-ComfyUI-Workflows** über `/sd` zugänglich macht (dynamisches Referenzbild + Prompt in benannte Knoten einspeisen).
- Which fidelity node is available in the user's ComfyUI install (IPAdapter vs. InstantID vs. FaceID vs. ReActor) — the extension should detect or let the user choose. / Welcher Treue-Knoten in der ComfyUI-Installation vorhanden ist — die Extension sollte erkennen oder wählen lassen.
- Storage size: reference images as data URLs in settings vs. relative paths returned by `/sd`. / Speichergröße: Referenzbilder als Daten-URL in den Settings vs. relative Pfade von `/sd`.
